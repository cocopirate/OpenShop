## 1. 背景与目标

消费者服务（consumer-service，:8001）负责消费者用户全生命周期管理，是面向 C 端用户的核心基础服务。其存在解决了以下问题：
- 统一管理消费者账号信息，为其他服务提供消费者数据查询能力
- 支持消费者注册流程，包括触发创建认证凭证（通知 auth-service）
- 管理消费者收货地址，支持多地址管理和默认地址设置
- 通过事件发布机制，驱动其他服务响应消费者数据变更

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 消费者 | 终端用户，通过 App/小程序使用平台服务 |
| app-bff | 面向 App 的聚合层，代理消费者操作请求 |
| auth-service | 消费者注册时由 consumer-service 通知创建认证凭证 |
| order-service | 下单时查询消费者收货地址信息 |

## 3. 功能需求

### 3.1 核心功能

- 消费者注册：创建消费者账号并发布注册事件，触发认证凭证创建
- 消费者信息查询：根据 ID 或手机号查询消费者基本信息
- 消费者信息更新：修改昵称、头像、性别等个人资料
- 地址管理：新增、编辑、删除、查询收货地址，支持设置默认地址
- 注销账号：标记消费者账号为注销状态，发布账号注销事件
- 事件发布：在关键操作（注册、信息变更、注销）时发布领域事件

### 3.2 功能详情

#### 消费者注册

- 输入：手机号、密码、昵称（可选）
- 处理：校验手机号唯一性 → 创建消费者记录 → 发布 `consumer.registered` 事件（由 auth-service 订阅，创建认证凭证）
- 输出：消费者 ID、手机号、创建时间
- 边界：手机号已注册返回 409；密码强度不满足要求返回 400

#### 消费者信息查询

- 输入：消费者 ID 或手机号
- 处理：查询消费者记录，过滤敏感字段
- 输出：消费者基本信息（ID、昵称、头像、手机号脱敏、注册时间、状态）
- 边界：消费者不存在返回 404；已注销账号返回 410

#### 地址管理

- 新增地址：输入收货人姓名、手机号、省市区、详细地址、邮编；处理：保存地址记录，若为第一条地址自动设为默认地址
- 更新地址：修改地址信息
- 删除地址：删除非默认地址；删除默认地址时自动将最近一条地址设为新默认地址
- 设置默认地址：将指定地址设为当前消费者的默认收货地址
- 地址列表：返回消费者所有地址，默认地址排在首位
- 边界：单个消费者最多保存 20 条地址

#### 事件发布

详见 [事件文档](events.md)，消费者服务发布以下事件：
- `consumer.registered`：消费者注册成功
- `consumer.profile_updated`：消费者个人信息变更
- `consumer.address_updated`：收货地址变更
- `consumer.deactivated`：消费者账号注销

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 查询接口 P99 < 100ms；注册接口 P99 < 500ms（含事件发布） |
| 安全 | 手机号在响应中脱敏显示；仅允许消费者本人或内部服务访问个人数据 |
| 幂等性 | 注册接口使用手机号唯一约束保证幂等；地址操作通过乐观锁防并发冲突 |
| 可用性 | 99.9% 月可用性；事件发布失败需重试，不阻塞主流程 |

## 5. 约束与依赖

| 依赖 | 说明 |
|------|------|
| PostgreSQL | 存储消费者账号和地址数据，数据模型详见 schema.md |
| RabbitMQ | 发布消费者领域事件 |
| auth-service | 消费者注册后由 auth-service 订阅事件创建认证凭证 |

主要约束：
- consumer-service 不直接调用 auth-service，通过事件解耦
- 消费者数据为其他服务的上游数据，变更需通过事件通知下游
- 端口 8001
