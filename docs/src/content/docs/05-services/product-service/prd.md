## 1. 背景与目标

商品服务（product-service，:8003）负责商品 SPU/SKU 的全生命周期管理、商品分类与属性管理，以及基于 Elasticsearch 的商品搜索，是平台商品供给侧的核心数据服务。其存在解决了以下问题：
- 提供统一的商品创建、编辑、上下架接口，作为商品数据的唯一权威来源
- 管理 SPU 与 SKU 的层级关系，支撑多规格商品场景
- 提供商品分类与属性体系，支持商品的结构化管理与搜索过滤
- 通过 Elasticsearch 提供高性能的全文搜索与多维度筛选能力，提升消费者购物体验

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 商家 | 创建、编辑、上下架自己店铺的商品（SPU/SKU） |
| 管理员 | 管理商品分类、属性模板；审核并强制下架违规商品 |
| 消费者 | 搜索、浏览商品列表与商品详情 |
| order-service / order-orchestration | 下单时查询 SKU 价格与状态，校验商品是否可购买 |
| inventory-service | 查询 SKU 信息以关联库存记录 |
| merchant-service | 提供店铺归属校验，product-service 通过其验证商家身份 |

## 3. 功能需求

### 3.1 核心功能

- SPU 管理：商家创建、编辑、查询 SPU（商品主体），设置商品名称、分类、描述及主图
- SKU 管理：在 SPU 下创建多规格 SKU，设置规格属性（如颜色、尺码）、价格
- 商品状态管理：维护 SPU 状态流转（DRAFT → ONLINE ↔ OFFLINE）
- 分类管理：管理员维护商品多级分类树，商家创建商品时选择叶子分类
- 属性管理：管理员定义分类下的属性模板（如"颜色"、"材质"）；商家填写 SKU 属性值
- 商品搜索：消费者通过关键词、分类、价格区间等条件搜索商品，结果由 Elasticsearch 提供
- 商品详情查询：返回 SPU 基本信息及其所有 SKU 列表
- 管理后台接口：管理员可跨商家查询所有商品，并执行强制下架操作

### 3.2 功能详情

#### SPU 管理

- 输入：商品名称（≤200 字符）、分类 ID、品牌 ID（可选）、描述、主图列表
- 处理：校验商家身份（merchant-service）→ 校验分类存在 → 创建 SPU 记录（状态 `DRAFT`）
- 输出：SPU ID、创建时间、当前状态
- 边界：商家状态非 `approved` 时不允许创建商品；SPU 名称不可为空

#### SKU 管理

- 输入：SPU ID、SKU 标题（≤200 字符）、规格属性（JSONB，如 `{"color":"red","size":"M"}`）、价格（≥0.01）
- 处理：校验 SPU 归属商家 → 创建 SKU 记录（默认状态 `ENABLE`）
- 输出：SKU ID、价格、状态、创建时间
- 边界：单个 SPU 下最多 200 个 SKU；价格精度为小数点后 2 位

#### 商品状态流转

```
DRAFT → ONLINE    （商家发布上架）
ONLINE → OFFLINE  （商家主动下架 或 管理员强制下架）
OFFLINE → ONLINE  （商家重新上架）
```

- 仅状态为 `ONLINE` 的商品对消费者可见
- 强制下架需记录操作人（管理员 ID）及下架原因

#### 分类管理

- 管理员创建/编辑/删除商品分类，支持多级（最多 3 级）树形结构
- 叶子分类可绑定属性模板，用于规范商家录入 SKU 属性
- 删除分类时，若分类下存在商品则不允许删除

#### 商品搜索

- 消费者输入关键词后，在 Elasticsearch 中检索 SPU 名称与描述
- 支持按分类 ID、价格区间过滤；支持按价格、销量、上架时间排序
- 搜索结果仅返回状态为 `ONLINE` 的商品
- 商品数据在 SPU 状态变更时同步更新至 Elasticsearch

#### SKU 状态管理

- 商家可启用（`ENABLE`）或禁用（`DISABLE`）单个 SKU
- 禁用的 SKU 不允许被下单

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 商品详情查询 P99 < 100ms；搜索接口 P99 < 300ms |
| 安全 | 商家只能创建和修改归属自己的商品；管理员接口需校验管理员身份 |
| 一致性 | SPU 状态变更后，Elasticsearch 索引最终一致性更新延迟 < 1s |
| 可用性 | 99.9% 月可用性；Elasticsearch 不可用时降级返回数据库查询结果 |
| 幂等性 | 创建 SPU/SKU 接口支持客户端幂等键（idempotency_key）防重复提交 |

## 5. 约束与依赖

| 依赖 | 说明 |
|------|------|
| PostgreSQL | 持久化 SPU、SKU、分类、属性数据 |
| Elasticsearch | 提供全文搜索与过滤能力 |
| merchant-service（:8002） | 校验商家身份与状态；验证店铺归属 |
| inventory-service | 查询 SKU 库存状态（下单前校验可用库存） |
| OSS | 存储商品图片，product-service 仅存储图片 URL |

主要约束：
- 端口 8003
- product-service 是商品数据的唯一持久化层，其他服务不直接读写商品表
- 商品只能在对应 SPU 状态为 `ONLINE` 且 SKU 状态为 `ENABLE` 时才允许下单
