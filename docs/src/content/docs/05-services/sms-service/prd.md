## 1. 背景与目标

短信服务（sms-service，:8010）是 OpenShop 平台的统一短信发送能力层，为各业务服务提供可靠、高可用的短信发送能力。其存在解决了以下问题：
- 平台各服务（order-service、auth-service 等）无需各自集成短信 SDK，统一通过 sms-service 发送短信
- 支持多供应商接入与自动切换，避免单一供应商故障导致短信发送中断
- 通过熔断器和限流机制，保护下游供应商 API 不被打爆
- 通过幂等性设计，防止因重试导致用户收到重复短信
- 支持多租户场景，不同业务方通过独立渠道路由和 API Key 隔离使用

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 业务服务（内部） | order-service、auth-service 等通过 API Key 调用发送短信 |
| 平台管理员 | 管理短信模板、渠道路由策略、API Key；查看发送记录和统计 |
| 短信供应商 | 阿里云、腾讯云、创蓝等，作为实际短信发送的第三方能力提供方 |

## 3. 功能需求

### 3.1 核心功能

- 短信发送：接收发送请求，通过模板渲染内容后调用供应商 API 发送
- 多供应商管理：支持阿里云、腾讯云、创蓝三个供应商，支持主备切换
- 熔断器：对每个供应商独立维护熔断状态，自动隔离故障供应商
- 限流：按 API Key/租户维度限制发送频率，防止滥用
- 幂等性控制：通过 `request_id` + Redis 防止重复发送
- 短信模板管理：维护短信模板，支持变量替换
- 多租户渠道路由：根据租户（API Key）配置，将请求路由到指定供应商渠道
- API Key 管理：为不同业务方创建和管理访问凭证

### 3.2 功能详情

#### 短信发送

- 输入：接收手机号、短信模板 code、模板变量、`request_id`（客户端生成的唯一 ID）
- 处理：
  1. 验证 API Key 有效性
  2. 查询 `request_id` 是否已在 Redis 中处理（幂等检查）
  3. 限流检查（按 API Key 和手机号双维度）
  4. 查询模板，渲染短信内容
  5. 根据租户路由策略选择目标供应商
  6. 调用供应商 API，通过熔断器保护
  7. 记录发送结果到数据库
  8. 将 `request_id` 写入 Redis（TTL 24 小时）
- 输出：发送结果（成功/失败）、供应商返回的消息 ID
- 边界：`request_id` 重复时直接返回上次结果；供应商熔断时自动切换备用供应商

#### 熔断器

- 每个供应商独立熔断状态：`closed`（正常）→ `open`（熔断中）→ `half-open`（探测恢复）
- 失败率阈值：连续失败 5 次或 60 秒内失败率 > 50% 时触发熔断
- 熔断持续时间：默认 60 秒
- `half-open` 状态下，允许少量请求通过以探测供应商恢复情况

#### 限流

- 全局限流：每个 API Key 每分钟最多发送 100 条
- 手机号限流：同一手机号每分钟最多收到 1 条，每天最多 10 条
- 超出限流返回 429 Too Many Requests

#### 短信模板管理

- 模板包含：模板 code（唯一）、模板名称、模板内容（含变量占位符 `${var}`）、适用供应商模板 ID 映射
- 支持创建、编辑、删除、查询模板
- 模板变量在发送时由请求方传入并替换

#### API Key 管理

- 为每个业务服务或租户分配唯一 API Key
- 支持设置每个 API Key 的限流配额和允许使用的模板范围
- 支持 API Key 的启用/禁用

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 短信发送接口 P99 < 2s（含供应商 API 调用）；幂等检查 P99 < 10ms |
| 幂等性 | 通过 `request_id` + Redis 保证同一请求不重复发送；Redis key TTL 24 小时 |
| 可靠性 | 主供应商失败时自动切换备用供应商；发送结果持久化，支持查询和对账 |
| 安全 | API Key 使用 HMAC 签名；管理接口需独立鉴权；手机号在日志中脱敏 |
| 可观测性 | 每次发送记录手机号（脱敏）、模板、供应商、结果、耗时；暴露供应商成功率指标 |

## 5. 约束与依赖

| 依赖 | 说明 |
|------|------|
| Redis | 存储幂等 key（request_id）、限流计数器、熔断器状态 |
| PostgreSQL | 存储短信模板、API Key、发送记录、渠道路由配置 |
| 阿里云短信 API | 供应商 API（dysmsapi.aliyuncs.com） |
| 腾讯云短信 API | 供应商 API（sms.tencentcloudapi.com） |
| 创蓝短信 API | 供应商 API（smssh1.253.com） |

主要约束：
- 端口 8010
- 所有供应商密钥通过环境变量注入，不硬编码
- 短信内容受各供应商审核约束，模板需提前在供应商后台报备
