## 1. 背景与目标

第三方服务集成（third-party-service）文档描述 OpenShop 平台对外部第三方能力的统一接入与管理策略。平台依赖多种第三方基础能力（短信、地图、搜索引擎、消息队列），通过统一的集成层降低各业务服务与外部依赖的耦合度，提升可替换性和可维护性。核心目标：
- 统一管理各类第三方服务的配置、密钥和供应商切换逻辑
- 为业务服务提供标准化的第三方能力调用接口，屏蔽不同供应商的 API 差异
- 通过熔断、重试、降级策略保障第三方依赖不可用时平台的整体稳定性

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 平台管理员 | 配置第三方服务密钥、切换供应商、查看调用统计和告警 |
| 业务服务（内部） | sms-service、product-service、location-service 等通过统一接口调用第三方能力 |
| 第三方供应商 | 阿里云、腾讯云、创蓝（短信）；高德、百度（地图）；Elasticsearch（搜索）；RabbitMQ（消息队列） |

## 3. 功能需求

### 3.1 核心功能

- 短信供应商管理：支持阿里云、腾讯云、创蓝三个短信供应商的配置与切换
- 地图服务集成：提供地址解析、逆地址解析、配送距离计算能力
- 搜索引擎集成：为 product-service 提供 Elasticsearch 商品全文检索能力
- 消息队列集成：RabbitMQ 作为统一事件总线，支持所有服务的异步事件传递
- 供应商配置管理：通过环境变量和管理 API 两种方式管理供应商配置
- 健康检查：定期检测第三方服务的连通性，发现异常及时告警

### 3.2 功能详情

#### 短信供应商管理

| 供应商 | 标识符 | API 地址 |
|--------|--------|---------|
| 阿里云短信 | `aliyun` | dysmsapi.aliyuncs.com |
| 腾讯云短信 | `tencent` | sms.tencentcloudapi.com |
| 创蓝短信 | `chuanglan` | smssh1.253.com |

- 主供应商通过环境变量 `SMS_PROVIDER` 配置，备用供应商通过 `SMS_PROVIDER_FALLBACK` 配置
- 支持运行时切换（通过管理 API `PUT /api/v1/admin/sms/config`），重启后恢复环境变量配置
- 切换操作需记录操作人和时间，支持回滚

#### 地图服务集成（location-service，:8008）

- 地址解析（Geocoding）：输入文字地址，输出经纬度坐标
- 逆地址解析（Reverse Geocoding）：输入经纬度，输出结构化地址
- 配送距离计算：输入起点和终点坐标，输出配送距离和预计时间
- 支持高德地图、百度地图，通过环境变量 `MAP_PROVIDER` 切换
- 边界：API 请求频率受供应商限额约束，需实现本地缓存（TTL 1 小时）减少外部调用

#### 搜索引擎集成（Elasticsearch）

- 为 product-service 提供商品全文检索能力
- 索引名：`products`；检索字段：`name`、`description`、`category`
- 支持中文分词（IK 分词器）
- 本地地址：http://localhost:9200；集群模式通过环境变量配置
- 边界：索引更新通过事件驱动异步进行，不影响主写入链路

#### 消息队列集成（RabbitMQ）

- 作为全平台统一事件总线，所有异步事件通过 RabbitMQ 传递
- Exchange 类型：Topic；消息持久化：开启
- 本地管理 UI：http://localhost:15672（用户名/密码：openshop/openshop123）
- 各服务的事件路由键命名规范：`{service}.{entity}.{action}`（如 `order.order.created`）
- 详见 [事件驱动架构](../../04-architecture/event-driven.md)

#### 健康检查

- 对所有第三方依赖（短信供应商、地图 API、Elasticsearch、RabbitMQ）定期发送心跳检测
- 检测结果暴露在 `/health` 端点，包含每个依赖的状态和最近一次检查时间
- 连续失败 3 次触发告警通知

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 可用性 | 单一第三方服务故障不影响平台其他功能；关键服务（短信）具备主备切换能力 |
| 安全 | 所有第三方 API 密钥通过环境变量或 K8s Secret 注入，禁止硬编码；密钥定期轮换 |
| 可观测性 | 记录每次第三方 API 调用的延迟、成功率；暴露 Prometheus 指标 |
| 降级 | 地图服务不可用时，配送距离计算返回默认值，不阻断下单流程 |

## 5. 约束与依赖

| 依赖 | 说明 |
|------|------|
| 阿里云 / 腾讯云 / 创蓝 | 短信发送，受各供应商 SLA 和限额约束 |
| 高德地图 / 百度地图 | 地图能力，受 API 配额约束 |
| Elasticsearch 8 | 商品搜索，需预装 IK 中文分词器插件 |
| RabbitMQ | 消息队列，平台所有异步事件的传输通道 |

主要约束：
- 第三方服务的 API 密钥和 Endpoint 配置必须通过环境变量，不得写入代码
- 供应商切换需经管理员审批，不允许业务服务自行切换
- RabbitMQ 为平台核心基础设施，需单独部署高可用集群
